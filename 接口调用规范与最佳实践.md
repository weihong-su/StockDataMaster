## 接口概览

### 核心接口

```python
from StockDataMaster import StockDataMaster

master = StockDataMaster()

# 1. K线数据接口（日线、分钟线）
df = master.get_kline(code, freq, start_date, end_date, count, adjust, use_cache)

# 2. 估值数据接口
df = master.get_valuation(code, start_date, end_date)

# 3. 实时Tick数据接口
tick = master.get_tick(code)

# 4. 股票名称查询
name = master.get_stock_name(code)
```

### 数据源优先级

| 数据类型 | 主数据源 | 备用数据源 | 说明 |
|---------|---------|-----------|------|
| **日K线** | Tushare | Mootdx, Baostock | 带双源校验缓存 |
| **分钟K线** | Mootdx | Tushare, Baostock | 不缓存 |
| **估值数据** | Baostock | - | 不缓存 |
| **实时Tick** | xtquant | - | 需QMT客户端 |

## 日K线数据接口

### 接口定义

```python
def get_kline(
    code: str,                    # 股票代码
    freq: str = 'd',              # 频率（'d'=日线）
    start_date: Optional[str] = None,  # 开始日期 'YYYY-MM-DD'
    end_date: Optional[str] = None,    # 结束日期 'YYYY-MM-DD'
    count: Optional[int] = None,       # 获取数量
    adjust: str = 'qfq',          # 复权类型（固定前复权）
    use_cache: bool = True        # 是否使用缓存
) -> pd.DataFrame
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 | 示例 |
|-----|------|------|------|------|
| `code` | str | ✅ | 股票代码（6位数字） | `'600519'`, `'000001'` |
| `freq` | str | ❌ | 频率，日线固定为 `'d'` | `'d'` |
| `start_date` | str | ❌ | 开始日期 | `'2025-01-01'` |
| `end_date` | str | ❌ | 结束日期 | `'2025-10-24'` |
| `count` | int | ❌ | 获取数量（最近N条） | `120` |
| `adjust` | str | ❌ | 复权类型，固定 `'qfq'` | `'qfq'` |
| `use_cache` | bool | ❌ | 是否使用缓存 | `True` |

### 请求模式

#### 模式1：按数量请求（推荐）
```python
# 获取最近120条日K线
df = master.get_kline('600519', freq='d', count=120)
```

**特点**：
- ✅ 最常用的请求方式
- ✅ 自动获取最新数据
- ✅ 盘中/盘后智能缓存
- ⚠️ 盘中时段：不缓存当日，每次获取实时数据
- ✅ 盘后时段：缓存当日收盘数据

#### 模式2：按日期范围请求（历史数据）
```python
# 获取指定日期范围的数据
df = master.get_kline('600519', freq='d',
                      start_date='2025-10-01',
                      end_date='2025-10-24')
```

**特点**：
- ✅ 适合历史数据分析
- ✅ **优化重点**：`end_date < 今天` 时直接使用缓存
- ✅ 盘中/盘后均可高效获取
- ✅ 历史数据永远不变，缓存命中率高

#### 模式3：按开始日期 + 数量
```python
# 从指定日期开始获取N条数据
df = master.get_kline('600519', freq='d',
                      start_date='2025-10-01',
                      count=20)
```

**特点**：
- ✅ 灵活的查询方式
- ⚠️ 实际行为取决于数据源实现
- ⚠️ 部分数据源可能不支持

### 数据长度限制

| 请求方式 | 推荐长度 | 最大长度 | 说明 |
|---------|---------|---------|------|
| **按数量** | 120条 | 无限制 | 推荐120条（缓存上限） |
| **按日期范围** | 任意 | 无限制 | 受数据源历史数据限制 |

**重要**：
- 日K线缓存最多保存 **120条/股票**（可配置 `cache.max_days_per_stock`）
- 超过120条的请求会从数据源获取，但只缓存最新120条
- 历史超长数据（如5年以上）建议分段获取

### 缓存行为（日K线）

#### 🕐 盘中时段（09:30-15:00）

| 请求类型 | 缓存行为 | 说明 |
|---------|---------|------|
| 最新N条 (`count`) | ❌ 不缓存当日 | 每次获取实时数据 |
| 历史范围 (`end_date < 今天`) | ✅ 使用缓存 | 历史数据不变 |
| 包含今天 (`end_date = 今天`) | ❌ 不缓存当日 | 重新获取 |

**示例**：
```python
# 盘中10:30请求
df1 = master.get_kline('600519', count=120)
# → 不缓存当日，返回实时数据 ✅

df2 = master.get_kline('600519', start_date='2025-10-01', end_date='2025-10-24')
# → 使用缓存，历史数据不变 ✅

df3 = master.get_kline('600519', start_date='2025-10-01', end_date='2025-10-25')
# → 不缓存2025-10-25，重新获取 ✅
```

#### 🕒 盘后时段（15:00-24:00）

| 请求类型 | 缓存行为 | 说明 |
|---------|---------|------|
| 首次请求 | ✅ 缓存当日收盘数据 | 数据已固定 |
| 再次请求 | ✅ 使用缓存 | 高效响应 |
| 历史范围 | ✅ 使用缓存 | 优先缓存 |

**示例**：
```python
# 盘后15:30首次请求
df1 = master.get_kline('600519', count=120)
# → 缓存当日收盘数据 ✅

# 盘后16:00再次请求
df2 = master.get_kline('600519', count=120)
# → 使用缓存，快速响应 ✅
```

#### 🗓️ 周末/节假日

| 请求类型 | 缓存行为 | 说明 |
|---------|---------|------|
| 任何请求 | ✅ 使用缓存 | 使用最新交易日数据 |

**示例**：
```python
# 周六请求
df = master.get_kline('600519', count=120)
# → 使用缓存，返回最后交易日（周五）数据 ✅
```

## 分钟K线数据接口

### 接口定义

```python
# 分钟线请求
df = master.get_kline(
    code='600519',
    freq='5m',      # '5m', '15m', '30m', '60m'
    count=240,      # 推荐240条（1天）
    use_cache=False # 分钟线不缓存
)
```

### 支持的频率

| 频率代码 | 说明 | 数据源 | 数据长度限制 |
|---------|------|--------|------------|
| `'5m'` | 5分钟线 | Mootdx | 最多800条 |
| `'15m'` | 15分钟线 | Mootdx | 最多800条 |
| `'30m'` | 30分钟线 | Mootdx | 最多800条 |
| `'60m'` | 60分钟线 | Mootdx | 最多800条 |

### 数据长度限制

#### Mootdx（主数据源）
- **最大长度**: 800条
- **推荐长度**:
  - 5分钟线：240条（1个交易日 = 4小时 × 60分钟 / 5分钟 = 48条）
  - 15分钟线：80条（1个交易日 = 48 / 3 = 16条）
  - 60分钟线：240条（约30个交易日）

#### Tushare（备用数据源）
- **最大长度**: 8000条
- **权限要求**: 需要足够的积分/权限点数

### 缓存行为（分钟线）

| 操作 | 缓存行为 | 说明 |
|------|---------|------|
| 请求分钟线 | ❌ **不缓存** | 分钟线数据量大，不缓存 |
| 每次请求 | 🔄 实时获取 | 从数据源获取 |

**原因**：
- 分钟线数据量大，缓存成本高
- 数据时效性要求高
- 主要用于实时分析和盘中监控

### 使用建议

#### ✅ 推荐用法
```python
# 获取最近1天的5分钟线
df = master.get_kline('600519', freq='5m', count=48)

# 获取最近5天的15分钟线
df = master.get_kline('600519', freq='15m', count=80)
```

#### ⚠️ 注意事项
```python
# ❌ 避免：请求超长数据
df = master.get_kline('600519', freq='5m', count=10000)
# → 超过Mootdx限制（800条），可能失败

# ❌ 避免：使用 use_cache=True
df = master.get_kline('600519', freq='5m', count=240, use_cache=True)
# → 分钟线不缓存，参数无效

# ✅ 正确：合理的数据量请求
df = master.get_kline('600519', freq='5m', count=240)
```

### 复权说明

⚠️ **重要**：Mootdx 的分钟线复权数据存在已知BUG
- 复权因子可能不准确
- 建议谨慎使用复权分钟线数据
- 如需精确复权数据，建议使用 Tushare

## 缓存机制说明

### 缓存策略总览

| 数据类型 | 缓存策略 | 缓存数据库 | 最大缓存 |
|---------|---------|-----------|---------|
| **日K线** | ✅ 智能缓存 | SQLite | 120条/股票 |
| **分钟K线** | ❌ 不缓存 | - | - |
| **估值数据** | ❌ 不缓存 | - | - |
| **Tick数据** | ❌ 不缓存 | - | - |

### 日K线缓存详解

#### 缓存条件

**必须同时满足**：
1. ✅ 频率为日线 (`freq='d'`)
2. ✅ `use_cache=True`（默认）
3. ✅ 数据通过双源校验（Tushare + Mootdx/Baostock）
4. ✅ 价格容差 ≤ ±0.01元 或 ±0.5%
5. ✅ 成交量容差 ≤ ±5%

#### 缓存写入时机

| 时段 | 缓存写入行为 | 说明 |
|------|------------|------|
| **盘中（< 15:00）** | ❌ 不缓存当日数据 | 跳过当日，只缓存历史 |
| **盘后（>= 15:00）** | ✅ 缓存当日收盘数据 | 收盘后数据已固定 |

#### 缓存读取逻辑

```
用户请求 → 检查缓存
    ↓
[1] end_date < 今天  →  使用缓存 ✅（历史数据不变）
    ↓
[2] 缓存最新日期 == 今天
    ├─ 盘中（< 15:00）→  不使用缓存（重新获取）
    └─ 盘后（>= 15:00）→  使用缓存
    ↓
[3] 缓存最新日期 == 最新交易日  →  使用缓存
    ↓
[4] 其他情况  →  不使用缓存（重新获取）
```

#### 缓存数据库

**位置**: `cache/kline_cache.db` (SQLite)

**表结构**:
```sql
CREATE TABLE kline_cache (
    code TEXT,           -- 股票代码
    date TEXT,           -- 交易日期
    open REAL,           -- 开盘价
    high REAL,           -- 最高价
    low REAL,            -- 最低价
    close REAL,          -- 收盘价
    volume REAL,         -- 成交量
    amount REAL,         -- 成交额
    source1 TEXT,        -- 主数据源
    source2 TEXT,        -- 校验数据源
    validated INTEGER,   -- 是否已校验
    created_at TEXT,     -- 创建时间
    updated_at TEXT,     -- 更新时间
    UNIQUE(code, date)
);
```

#### 缓存管理

```python
# 查看缓存统计
stats = master.get_cache_statistics()
print(stats)
# {
#     'enabled': True,
#     'total_records': 1200,

#     'db_size_mb': 0.15,
#     'db_path': 'cache/kline_cache.db'
# }
```

### 场景3：回测（大量历史数据）

**需求**：获取5年历史数据进行策略回测

**推荐方案**：
```python
import pandas as pd
from datetime import datetime, timedelta

# 分段获取（避免单次请求过长）
def get_long_history(code, years=5):
    all_data = []
    end_date = datetime.now()
    start_date = end_date - timedelta(days=years*365)

    # 每次获取120天（缓存上限）
    current = end_date
    while current > start_date:
        segment_end = current.strftime('%Y-%m-%d')
        segment_start = (current - timedelta(days=120)).strftime('%Y-%m-%d')

        df = master.get_kline(code, freq='d',
                              start_date=segment_start,
                              end_date=segment_end)
        if df is not None and not df.empty:
            all_data.append(df)

        current -= timedelta(days=120)

    return pd.concat(all_data, ignore_index=True)

# 使用
df_5y = get_long_history('600519', years=5)
```

**特点**：
- ✅ 分段获取，充分利用缓存
- ✅ 避免单次请求过长导致的超时
- ✅ 可复用，多次运行命中缓存

### 场景4：多股票批量查询

**需求**：获取多只股票的最新数据

**推荐方案**：
```python
codes = ['600519', '000001', '000858', '601318']

def get_latest_batch(codes):
    results = {}
    for code in codes:
        df = master.get_kline(code, freq='d', count=1)
        if df is not None and not df.empty:
            results[code] = {
                'date': df['date'].iloc[-1],
                'close': df['close'].iloc[-1],
                'volume': df['volume'].iloc[-1]
            }
    return results

# 使用
latest_prices = get_latest_batch(codes)
```

**优化建议**：
```python
# 使用多线程并发请求（注意API限流）
from concurrent.futures import ThreadPoolExecutor

def get_latest_batch_parallel(codes, max_workers=3):
    def fetch_one(code):
        df = master.get_kline(code, freq='d', count=1)
        if df is not None and not df.empty:
            return code, df
        return code, None

    results = {}
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        futures = [executor.submit(fetch_one, code) for code in codes]
        for future in futures:
            code, df = future.result()
            if df is not None:
                results[code] = df
    return results
```

**注意事项**：
- ⚠️ 控制并发数量（推荐 ≤ 3）
- ⚠️ 注意API限流（Tushare有调用频率限制）
- ✅ 盘后批量查询命中缓存，效率高

### 场景5：盘后数据更新

**需求**：每天收盘后更新所有持仓股票数据

**推荐方案**：
```python
import schedule
import time

def update_daily_data():
    """每天15:30更新数据"""
    codes = ['600519', '000001', '000858']  # 持仓股票列表

    for code in codes:
        # 获取最新1条数据，触发缓存写入
        df = master.get_kline(code, freq='d', count=1)
        print(f"{code} 更新完成: {df['date'].iloc[-1]}")

# 定时任务：每天15:30执行
schedule.every().day.at("15:30").do(update_daily_data)

while True:
    schedule.run_pending()
    time.sleep(60)
```

**特点**：
- ✅ 盘后15:00后数据已固定
- ✅ 首次请求缓存当日收盘数据
- ✅ 后续查询命中缓存，快速响应

## 约束条件与限制

### 数据源限制

#### Tushare
- **调用频率**: 根据积分等级限制（如 200次/分钟）
- **数据长度**: 日线无限制，分钟线最多8000条
- **权限要求**: 分钟线需要足够的积分点数
- **Token配置**: 需在 `config.json` 中配置有效Token

#### Mootdx
- **数据长度**: 日线无限制，分钟线最多800条
- **复权问题**: ⚠️ 分钟线复权数据有BUG，谨慎使用
- **网络要求**: 需要稳定的网络连接

#### Baostock
- **性能**: 免费但速度较慢
- **适用场景**: 作为备用数据源
- **数据范围**: 支持日线、分钟线、估值数据

#### xtquant
- **依赖**: 需要本地运行 QMT 客户端
- **使用场景**: 仅用于实时Tick数据
- **限制**: 如果QMT未运行，自动禁用

### 缓存限制

| 项目 | 限制值 | 配置项 |
|------|-------|--------|
| **单只股票最大缓存** | 120条 | `cache.max_days_per_stock` |
| **价格容差（绝对）** | ±0.01元 | `cache.validation.price_tolerance_abs` |
| **价格容差（相对）** | ±0.5% | `cache.validation.price_tolerance_pct` |
| **成交量容差** | ±5% | `cache.validation.volume_tolerance_pct` |
| **缓存数据库路径** | `cache/kline_cache.db` | `cache.db_path` |

### 系统限制

#### 股票代码格式
```python
# ✅ 支持的格式
'600519'      # 6位数字
'sh.600519'   # 带市场前缀（内部会转换）
'sz.000001'   # 带市场前缀

# ❌ 不支持的格式
'600519.SH'   # 后缀格式
'SH600519'    # 连写格式
```

#### 日期格式
```python
# ✅ 正确格式
'2025-10-24'  # YYYY-MM-DD

# ❌ 错误格式
'2025/10/24'  # 斜杠分隔
'20251024'    # 无分隔符
'24-10-2025'  # 非标准顺序
```

#### 复权类型
```python
# ✅ 唯一支持
adjust='qfq'  # 前复权

# ❌ 不支持
adjust='hfq'  # 后复权
adjust=None   # 不复权
```

**说明**：系统统一使用前复权（qfq），确保数据一致性。

## 常见问题 FAQ

### Q1: 盘中请求数据为什么不使用缓存？

**A**: 盘中时段（09:30-15:00），当日K线数据是动态变化的。为确保数据实时性，系统不缓存当日数据，每次请求都从数据源获取最新数据。盘后时段（15:00后），数据已固定，会正常缓存。

---

### Q2: 如何判断返回的数据是否来自缓存？

**A**: 检查 `df.attrs['source']` 属性：
```python
df = master.get_kline('600519', count=120)
source = df.attrs.get('source', 'unknown')

if source == 'cache':
    print("数据来自缓存")
else:
    print(f"数据来自数据源: {source}")
```

---

### Q3: 为什么分钟线不缓存？

**A**: 原因：
1. 分钟线数据量大（1天48条×多股票），缓存成本高
2. 分钟线主要用于实时分析，时效性要求高
3. 数据源限制（Mootdx最多800条），不适合长期缓存

---

### Q4: 如何提高缓存命中率？

**A**: 最佳实践：
```python
# ✅ 好：请求历史数据（充分利用缓存）
df = master.get_kline('600519', start_date='2025-10-01', end_date='2025-10-24')

# ✅ 好：盘后请求最新数据（使用缓存）
df = master.get_kline('600519', count=120)  # 盘后时段

# ⚠️ 一般：盘中请求最新数据（不使用缓存）
df = master.get_kline('600519', count=120)  # 盘中时段

# ❌ 差：禁用缓存
df = master.get_kline('600519', count=120, use_cache=False)
```

---

### Q5: 请求超过120条数据会怎样？

**A**:
- 系统会从数据源获取完整数据
- 但只缓存最新120条
- 超过120条的历史数据不缓存
- 建议：如需超长数据，使用日期范围请求，并分段获取

---

### Q6: Tushare Token 如何配置？

**A**: 在 `config.json` 中配置：
```json
{
  "data_sources": {
    "tushare": {
      "enabled": true,
      "token": "你的Tushare Token"
    }
  }
}
```

获取Token：访问 https://tushare.pro/ 注册并获取

---

### Q7: 双源校验是什么？

**A**: 为确保数据准确性，日K线缓存采用双源校验机制：
1. 主数据源（Tushare）获取数据
2. 校验数据源（Mootdx或Baostock）获取相同数据
3. 对比两份数据，检查价格和成交量差异
4. **只有校验通过的数据才会缓存**

容差标准：
- 价格：±0.01元 或 ±0.5%
- 成交量：±5%

---

### Q8: 如何清理缓存？

**A**:
```python
# 方法1：清理超过120天的旧数据
master.cleanup_cache(days=120)

# 方法2：手动删除缓存数据库
import os
os.remove('cache/kline_cache.db')
# 下次启动时会自动重建
```

---

### Q9: 周末请求数据返回什么？

**A**: 周末/节假日请求数据时：
- 返回最后一个交易日（通常是上周五）的数据
- 使用缓存，不会调用数据源API
- 最新数据日期会早于当前日期

```python
# 周六请求
df = master.get_kline('600519', count=120)
print(df['date'].iloc[-1])  # 输出: 2025-10-24（周五）
```
